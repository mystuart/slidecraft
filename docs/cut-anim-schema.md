# cut-anim 组件 Schema 草案（v0.1）

> 目标：在课件里**动画展示**「从三棱柱切下一刀得到三棱锥」的过程，让学员**亲眼看到**三棱锥是「切下」而不是「凭空冒出来的」 —— 这是立体几何求三棱锥体积「割补法 / 等体积法」教学的核心可视化。
>
> 适用场景：立体几何大题里求三棱锥体积 —— 学员的卡点往往是「为什么可以用 V_keep / V_total 这个比例」—— 动画能把比例关系变成肉眼可见的几何动作。

## 字段一览

| 字段 | 类型 | 必填 | 默认 | 说明 |
|------|------|------|------|------|
| `id` | string | - | 自动生成 | 组件根 DOM ID |
| `title` | string | - | - | 卡片小标题 |
| `caption` | string | - | - | 卡片下方说明文字 |
| `linkedGeometry3d` | string | ✅ | - | 联动的 geometry-3d 实例 ID（顶点源） |
| `keepVertices` | string[] | ✅（4 个） | - | 切下保留的 4 个顶点（构成目标三棱锥） |
| `cutPlane` | object | ✅ | - | `{type: "plane-through-points", points: [lbl1, lbl2, lbl3]}` 切割平面 |
| `duration` | number | - | `1500` | 动画时长（ms） |
| `easing` | string | - | `'easeInOutQuad'` | 缓动函数（可选 `linear` / `easeOutCubic`） |
| `playButton` | bool | - | `true` | 是否显示播放按钮；`false` = 加载即自动播一次 |
| `loop` | bool | - | `true` | 动画结束后保持显示「保留四面体」还是隐藏（false = 回到 0） |
| `camera` | object | - | 自动框 | `{position:[x,y,z], target:[x,y,z]}` |
| `showVolumeHint` | bool | - | `true` | 是否显示「保留 / 整体 / 比例」数值提示 |

## keepVertices 字段

`keepVertices` 是 **4 个 label 名**（如 `["A", "P", "C", "B₁"]`）—— 表示切下保留的 4 个顶点，必须构成一个三棱锥（4 顶点不共面）。

## cutPlane 字段

`cutPlane` 是切割平面的描述。**当前 v0.1 只支持** `plane-through-points` 类型：

```jsonc
{
  "type": "plane-through-points",
  "points": ["A", "P", "C"]   // 切割平面过的 3 个 label
}
```

切平面的法向 = `(points[1] - points[0]) × (points[2] - points[0])`，可视化时把这 3 点构成的三角形作为切口淡显示。

**注意**：v0.1 的切平面是「退化三角形」（3 个顶点 = 平面定义，没有 4 个顶点扩展成大三角）。后续 v0.2 可考虑用平面定义自动扩展切面大小。

## 动画设计

不引入 Three.js 的 `clippingPlane`（与 slider/derived 联动兼容性差），改用「透明度动画」：

| 进度 | 完整三棱柱 | 保留四面体 | 切平面 |
|------|------------|------------|--------|
| 0.0  | 完全不透明 (1.0) | 完全透明 (0.0) | 透明 |
| 0.5  | 半透 (0.65) | 半透 (0.43) | 半透 (0.4) 达到峰值 |
| 1.0  | 淡化 (0.3) | 不透明 (0.85) | 淡出回 0 |

学员看到的：
1. **进度 0**：只看到完整三棱柱（标准几何体）
2. **点击播放**（或自动播放）：保留四面体从无到有「生长」出来，三棱柱逐渐褪色
3. **进度 0.5**：两者并存 —— 学员能直接看到「保留部分」和「完整几何体」的空间关系
4. **进度 1.0**：保留四面体高亮，三棱柱淡到背景 —— 「切下完成」

## 联动规则

- 组件**每帧**调用 `api.getLabelPos(label)` 拉取最新顶点位置
- 联动源 `linkedGeometry3d` 通过 `slider` 改 `P` 位置时：
  1. `slider.drives` 调 `api.setLabelPos("P", [...])`
  2. 下一帧本组件拉取到新 `P` 位置
  3. 保留四面体 / 切平面 / 完整三棱柱同步重画
- 联动源晚于本组件就绪时：组件初始化后 200ms / 800ms 各重拉一次

## 体积提示

`showVolumeHint: true`（默认）时，控件栏下方显示：

> 保留四面体 ≈ 0.5833 · 整体三棱柱 = 2.0 · 比例 ≈ 0.2917 : 1

- **保留体积**：4 个 keepVertices 构成的体积（六面体体积公式）
- **整体体积**：`底面积 × 高`（用向量叉积算底面积，用 B₁ 到平面 ABC 的距离算高）
- **比例**：保留 / 整体

学员看到数字的同时看到几何动作，能把抽象的「比例」直接挂钩到空间位置。

## 联动示例：从三棱柱切出三棱锥 B₁-APC

**geometry-3d 声明**（顶点源）：

```jsonc
{
  "id": "prism-2024-xianyang-q2",
  "geometry": "triangular-prism",
  "vertices": [[0,2,0], [0,0,0], [2,0,0], [0,2,2], [0,0,2], [2,0,2]],
  "labels": [
    ["A",  [0, 2, 0]],
    ["B",  [0, 0, 0]],
    ["C",  [2, 0, 0]],
    ["A₁", [0, 2, 2]],
    ["B₁", [0, 0, 2]],
    ["C₁", [2, 0, 2]],
    ["P",  [1.5, 2, 2]]
  ]
}
```

**cut-anim 声明**：

```jsonc
{
  "id": "cut-b1apc",
  "title": "剖切动画：从三棱柱切出三棱锥 B₁-APC",
  "linkedGeometry3d": "prism-2024-xianyang-q2",
  "keepVertices": ["A", "P", "C", "B₁"],
  "cutPlane": {
    "type": "plane-through-points",
    "points": ["A", "P", "C"]
  },
  "duration": 1800,
  "easing": "easeInOutQuad",
  "caption": "切下部分 = 三棱锥 B₁-APC；剩余部分 = 三棱柱 APC-A₁B₁C₁ 减掉 B₁-APC 的剩余。比例 ≈ 7/12 : 5/12（与 P 位置无关，因为 V_keep 只跟三角形 APC 面积 × B₁ 到面 APC 距离有关）。"
}
```

学员看到的：
- 完整三棱柱（淡蓝） + 保留三棱锥（橙色高亮）的叠加
- 点播放 → 保留三棱锥从透明淡入到不透明，三棱柱从 1.0 淡到 0.3
- 切平面（A-P-C 三角形）淡入到 0.4 再淡出，提示「这就是切口」
- 拖动上方 slider 改 P 位置 → 切面跟着转，但保留四面体比例关系不变

## v0.1 实现边界

- **不引入 Three.js `clippingPlane`**（兼容性差 + 与 slider/derived 联动风险高）
- **不做物理位移**（如让切下部分整体平移出三棱柱）
- **不做切面纹路**（只展示几何，不展示平面纹理）
- **仅支持 `plane-through-points`**（不支持平面方程 / 法线直接给定）
- **完整几何体的顶点必须全部在源 geometry-3d 的 labels 里声明**（A, B, C, A₁, B₁, C₁ 这 6 个）
- **`keepVertices` 必须正好 4 项**

## 已知问题

- 「切下部分」和「完整几何体」在同一个 canvas 里叠加，3D 拖动时学员可能误判空间关系 —— 建议配合「旋转 + 暂停」教学法使用
- 比例提示的「切下 / 剩余」用体积公式估算，未做精确 mesh-level 切割 —— 误差量级在 $10^{-6}$
- linkedGeometry3d 实例未就绪时组件 fallback：等 200ms / 800ms 重拉；如仍失败则一直 0% 不动
- 切平面是「3 点退化三角形」，不是真正的无限大平面 —— 视觉上能识别切口位置，但没有切面阴影 / 投影效果
