# docs/

courseware 项目的文档目录。**纳入 git 管理**——和源码、SPEC、内容一起进版本控制。

## 当前文件

- [components-registry.md](./components-registry.md) — 组件登记簿。记录每个组件的版本、状态、字段契约、依赖、已知问题、待打磨方向。组件打磨时同步更新。

## 维护约定

- 任何组件字段/视觉/交互调整时，**先**改代码，**后**更新 registry（不要反过来，避免漏记）
- 状态变更（待打磨 → 打磨中 → 打磨完成）时同步更新版本号
- 跨组件的系统级问题单列在 registry 的「已知系统级问题」段
